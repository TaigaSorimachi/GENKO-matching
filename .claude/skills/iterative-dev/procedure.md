計画された各タスクを実行する際の動作手順：
- 関連する @rules/frontend/ と @rules/backend/ 配下のルールを再確認する。
- `edit` を使用して、コードを小さな単位で段階的に実装する。
- 実装直後に `shell` で `npx tsc --noEmit` を実行し、型安全性を確認する。
- 対応するテストファイルが存在する場合は、`npm test <path>` を実行する。
- コマンドが失敗した場合は、`read` でエラー出力を確認し、コードを修正して、合格するまでリトライする。
- 各タスクが成功するたびに `git commit` を行い、進捗を確定させる。
